Pintos Projects 1-3

Each pintos project was directed to improving certain aspects to the existing OS, pintOS. Implemented a multilevel feedback queue scheduler which managed threads (priority donations, time quantumâ€™s, etc.). Added the ability for system calls in the OS, the implementation handled pushing arguments on to the stack, verifying pointers, and executing the indicated system call by system call number. Doodle Jump (Assembly)

Doodle jump is a replication of the classic phone game doodle jump. This replication was created purely in assembly code with the help of bitmaps. It functioned just as the original game did, a player would jump from platform to platform and kept this up for as long as they could until a platform was missed. Players could keep track of their high score and try their best to beat it time after time.

The Six Degrees of Kevin Bacon

This project was created according to the classic theory/game: The Six Degrees of Kevin Bacon. It used Neo4j's graph data base to create a graph connecting actors to movies that they were a part of. Multiple API endpoints were created to add actors, add movies, create relationships, and most importantly compute the "Bacon number" for a specified actor. The project was created using java with the help of Dagger and other libraries.

Transit App GUI

This application was created using javaFX. It automated bussing routes for bus riders. The application had two sides, admins and users. Admins had the ability to add routes, delete route, manage accounts, and pull up user information. Users could sign up or log in at the main screen, and once logged into their account they could look at their trip history, add money to their account, and tap on/off busses. 

XMODEM Server

The XMODEM server was created in C. The purpose of the server was for clients to send files over the internet. The server used the TCP protocall and could manage parallel clients by using states. 

Compression and Decompression algorithm

Zoomer

Zoomer is a bask end ride sharing service that connects people who need rides to drivers. Users were connected to the nearest drivers basedoff of distance and location. The back end service was made using MongoDB, PostgreSQL, and Neo4j. Each data base was used for their strengths in specific tasks. AnAPIgateway manages each request from our clients and routes them to their respective microservice. The back end service was made in java with the help of Apache maven libraries.
